# Complexity Rating

Rate the complexity of a task based on its subproblems.

## Instructions

Given the subproblems (or a prompt to decompose), rate the complexity:

### Easy
- 1-2 subproblems
- Straightforward, single-app actions
- No reasoning or decision-making required
- Example: "Create a new text file with specific content"

### Medium
- 3-5 subproblems
- May involve multiple apps or multi-step workflows
- Some reasoning required (e.g., formatting, calculations)
- Example: "Create a spreadsheet with formulas and conditional formatting"

### Hard
- 6+ subproblems
- Cross-app workflows, complex reasoning
- Multi-step with dependencies between steps
- Example: "Research data from a website, organize in a spreadsheet, create a presentation summarizing findings"

### Output:

```
Complexity: [Easy/Medium/Hard]
Subproblem count: [N]
Reasoning: [1-2 sentence justification]
```
